# 🚀 超简化部署指南 - 只需 4 步！

## 第 1 步：创建仓库（1 分钟）

打开这个链接：
```
https://github.com/new
```

填写：
- **Repository name**: `japan-trip-2026`
- 其他都不用管，直接点击 **Create repository** 按钮

---

## 第 2 步：获取密钥（2 分钟）

打开这个链接：
```
https://github.com/settings/tokens
```

操作：
1. 点击 "Generate new token" → "Generate new token (classic)"
2. 填写：
   - Note: `Deploy`
   - Expiration: `90 days`
   - Select scopes: 勾选 **repo**（展开后全选）
3. 点击 **Generate token**
4. **复制显示的密钥**（只显示一次！）

⚠️ **把这个密钥保存到记事本，等下要用**

---

## 第 3 步：推送代码（2 分钟）

**双击运行**这个文件：
```
c:\Users\70927\WorkBuddy\20260407093217\deploy.bat
```

脚本运行后会提示输入：
- **Username**: `chenniyang666`
- **Password**: **粘贴第 2 步复制的密钥**

---

## 第 4 步：开启网站（1 分钟）

推送成功后，打开这个链接：
```
https://github.com/chenniyang666/japan-trip-2026/settings/pages
```

设置：
- **Source**: 选择 `Deploy from a branch`
- **Branch**: 选择 `main`
- **Directory**: 选择 `/ (root)`
- 点击 **Save**

等待 1-2 分钟，刷新页面，就会显示：
```
🎉 Your site is live at:
https://chenniyang666.github.io/japan-trip-2026/
```

---

## ✅ 完成！

打开浏览器访问：
```
https://chenniyang666.github.io/japan-trip-2026/
```

## 🆘 遇到问题？

### 问题 1：deploy.bat 打不开
→ 右键点击文件，选择"以管理员身份运行"

### 问题 2：推送失败
→ 检查密钥是否正确复制，确保没有多余的空格

### 问题 3：网页显示 404
→ 等待 2-3 分钟再刷新，GitHub Pages 需要时间部署

### 问题 4：不知道 GitHub 密码是什么
→ 不用密码！用第 2 步获取的密钥 Token
